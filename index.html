Git Summary

1. Git Start

2. Git first command git init

3. Two stages of files in got repo: Tracked and Untracked

4. Tracked files are the file that git knows about and are added in the git repo

5. Untracked files are the files that are in the current working directory but are not added in the repository

6. git status : command shows the current status of the files which are both in directory as well as in the staging area. It also provides the status of the files that are being tracked by git

7. git status --short : to check the status of the repo in a compact/short way

8. git add fileName : adds the file to the staging area after a milestone is reached during a certain point in the completion of the code git add --all: for staging multiple files git add --A: for staging multiple files

9. git commit: command for saving the staged code

10. git commit --all: command for commiting multiple files in the current working directory

11. git commit -m"meaninful message to the code file while commiting"

12.git commit -a: command for directly commiting the file without moving the file to staging area for a small changes in the code files

13. Hello World

14. Always required to stage before commit

15.After commiting, whenever a change is done, it is required to stage and again commit

16. git --help for all the help available

17. git command -help fpr specific comand help. Like :- git add -help

GitHub

first create an account on GitHub

Then go to github and create repository and set it as public

Then go to gitbash application and use command : git remote add origin URL : To add a remote repository to the local git repository

git push --set-upstream origin master : This is to push the master branch to the origin url and set it as default remote branch

git push origin : to save the changes to the git remote repository from master repository file

Pull: Whenever a team is working on a project, it is required that everyone stays upto date

Therefore this can be done via pull. Pull is a combination of two commands: Fetch and merge

Fetch is used to view the changes in the tracked brach/repo

git diff origin/master : to view the difference made in the repo

git merge origin/master : it is used to merge the changes

after git merge check the status and commit the changes

git pull is a combination of fetch and merge. If you want to directly update the changes to the local repo with fetch and merge.

git push: it to push the changes from local repo to github url/remote github
